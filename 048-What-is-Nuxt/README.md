# Introduction to Nuxt

🚀 В данном разделе рассмотрены основы работы с Nuxt — фреймворком, построенным поверх Vue и предназначенным для создания более производительных и удобных для поисковых систем веб-приложений.

## Что такое Nuxt?

Nuxt расширяет возможности Vue и решает ряд проблем, характерных для обычных SPA-приложений (Single Page Applications). Одним из главных преимуществ является улучшенная поддержка SEO за счёт серверного рендеринга и генерации статических страниц.

## Основные концепции

### Server-Side Rendering (SSR)

- Страница формируется на сервере при каждом запросе пользователя.
- Пользователь получает готовый HTML.
- Контент всегда актуален.
- Подходит для динамических сайтов.

### Static Site Generation (SSG)

- Страницы генерируются заранее во время сборки проекта.
- Сервер просто отдаёт готовые HTML-файлы.
- Более высокая производительность по сравнению с SSR.
- Хорошо подходит для блогов, документации и контентных сайтов.

## Преимущества Nuxt

- Минимальная начальная настройка проекта.
- Встроенная поддержка Vue Router.
- Автоматическая генерация маршрутов на основе структуры папок.
- Поддержка пользовательских макетов страниц (Layouts).
- Удобная работа с серверными API.
- Поддержка Composition API.
- Улучшенная SEO-оптимизация по сравнению с обычными SPA.

## Автоматическая маршрутизация

Вместо ручной настройки маршрутов Nuxt использует структуру папки `pages`.

Например:

```text
pages/
├── index.vue
└── about.vue
```

Автоматически создаст маршруты:

```text
/
about
```

Таким образом, создание новых страниц сводится к добавлению файлов и директорий.
