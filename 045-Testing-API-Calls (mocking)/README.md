# Testing API Calls (Mocking)

В этом уроке было рассмотрено тестирование компонентов, которые выполняют HTTP-запросы к внешним API. Для этого использовалась техника **mocking** — подмена реальных сетевых запросов заранее подготовленными данными.

## Что такое Mocking

Mocking позволяет имитировать работу внешних сервисов во время тестирования. Вместо настоящего запроса в интернет тест получает заранее подготовленный ответ. Это делает тесты:

- быстрее;
- стабильнее;
- независимыми от состояния сервера и сети;
- более предсказуемыми.

## Что было реализовано

### Компонент PostCard

Создан компонент `PostCard.vue`, который:

- выполняет запрос через `axios`;
- отображает сообщение загрузки во время ожидания ответа;
- показывает данные поста после успешного запроса;
- отображает сообщение об ошибке при неудачном запросе.

### Тест успешного запроса

В тесте был использован:

```js
vi.spyOn(axios, "get").mockResolvedValueOnce(...)
```

Такой мок подменяет настоящий HTTP-запрос и возвращает подготовленные данные поста.

После завершения запроса с помощью:

```js
await flushPromises();
```

проверяется корректность отображения полученных данных.

### Тест обработки ошибки

Для проверки обработки ошибок использовался:

```js
vi.spyOn(axios, "get").mockRejectedValueOnce(...)
```

Этот мок имитирует неудачный запрос и позволяет проверить, что компонент корректно показывает сообщение об ошибке пользователю.

## Новые инструменты

### vi.spyOn()

Позволяет следить за вызовами существующего метода и при необходимости подменять его результат.

### mockResolvedValueOnce()

Возвращает успешный результат один раз.

### mockRejectedValueOnce()

Возвращает ошибку один раз.

### flushPromises()

Ожидает завершения всех асинхронных операций перед выполнением проверок.

## Итог

В результате урока были написаны тесты для проверки как успешного получения данных с API, так и обработки ошибок. Благодаря мокам удалось протестировать компонент без выполнения реальных сетевых запросов.
